Core Architecture

Understanding Full Stack

Explore how client devices communicate across networks to server infrastructures to execute code and retrieve database elements.

1. Frontend Layer

The presentation layer. Interpreted by client web browsers (Chrome, Safari, Firefox). Governed by HTML5 for structural layout, CSS3 for stylistic elements and animations, and JavaScript for client-side behaviors.

2. Backend Layer

The application business logic layer. Runs on remote servers (e.g. Node.js, Python, PHP). It handles requests dispatched by clients, manages authentication, process calculations, and reads/writes databases.

3. Database Layer

The data persistence layer. Stores student credentials, catalog assets, log registries. Implemented using SQL systems (MySQL, PostgreSQL) or NoSQL databases (MongoDB, Redis) for optimized indexing.

4. API Interface

Application Programming Interfaces. Communication protocols (REST, GraphQL, WebSockets) allowing client frontend apps to seamlessly request and load payload resources from backend layers.

Client-Server Communication & Web Flow

When a student accesses a web page, the browser issues an HTTP GET Request over the network via IP routing. The destination server interprets the request header, performs authorization and routing, runs target scripts, queries databases, and outputs response blocks.

For modern single-page designs or interactive portals, asynchronous requests utilize the fetch API. This lets the web page request specific data points dynamically without redrawing the entire screen layout.

Engineering Concept: HTTP Protocols

HTTP (Hypertext Transfer Protocol) is stateless, request-response based. Actions are map-associated with methods: GET (read), POST (create), PUT (replace), and DELETE (remove).

Static Web Pages vs. Dynamic Web Apps

Static Websites

Static web pages deliver identical, pre-built files (HTML, CSS, assets) to every user. These files are loaded directly from the hosting disk and bypass server processing databases.

  • Performance: Ultra-fast because they are cached via CDNs.
  • Complexity: Low. Built purely with HTML, CSS, and basic style wrappers.
  • Use Case: Portfolios, documentation, landing sheets, company cards.
  • State: All users view exactly the same content unless manually updated.

Dynamic Web Applications

Dynamic web applications process resources on-the-fly. The server constructs unique HTML blocks or returns raw JSON payloads based on data queries, logged-in session parameters, and user permissions.

  • Interaction: Tailored specifically to the active user (dashboard feeds).
  • Processing: High. Involves server scripts and queries.
  • Use Case: Social media feeds, e-commerce, banking dashboards, collaborative documents.
  • State: Changes in real time based on active database operations.

// Fetch student data asynchronously from backend server
function loadStudentData() {
    fetch('/api/student/ashim')
        .then(response => response.json())
        .then(data => {
            document.getElementById('student-name').textContent = data.name;
            console.log('Data Loaded Successfully', data);
        })
        .catch(err => {
            console.error('Error fetching details:', err);
        });
}
